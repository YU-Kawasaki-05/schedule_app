# Docs Research

## Goal
Answer documentation-sensitive questions using primary sources and exact references.

## Workflow
1. Identify the exact API, framework, or version in question.
2. Prefer primary docs over blog posts or forum replies.
3. Note any version-sensitive or experimental behavior.
4. Summarize the answer in plain language.
5. Call out uncertainty if the docs are ambiguous.

## Output
Return:
- Answer
- Source hierarchy used
- Version / scope caveats
- Recommended next action if the docs are ambiguous
