# UI Design

## 基本方針

- `DESIGN.md` のトークンをCSS variablesとして実装に反映する。
- 回答者画面はスマホファースト。
- 管理者画面はPCでの一覧性を重視しつつ、スマホでも最低限使えるようにする。
- shadcn/ui は初期導入しない。

## コンポーネント方針

- `src/components/ui/` に Button、Input、Textarea、Badge、Panel などの小さな部品を置く。
- フォームラベルは常に表示する。
- エラーは入力欄の近くに表示する。
- 重要な状態は色だけでなくテキストでも表す。

## 回答者画面

- 候補日時はカード形式。
- タップで `行けない` / `行ける` を切り替える。
- 選択数を送信ボタン文言に反映する。
- 送信ボタンはモバイルで画面下部に固定する。

## 管理者画面

- 詳細画面の上部に最も集まりやすい候補を表示する。
- 回答マトリクスは横スクロールを許可する。
- 削除などの破壊的操作は危険色と確認導線で区別する。
