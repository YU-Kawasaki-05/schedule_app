# Process Overview

## 完成までの工程

1. 要件と設計を分割docsへ整理する。
2. Next.jsアプリ基盤を作る。
3. デザインシステムを実装に反映する。
4. Supabase schema、RLS、環境変数を整える。
5. 管理者認証を作る。
6. 管理者イベント管理を作る。
7. 公開回答と回答編集を作る。
8. 回答集計を作る。
9. エラー、空状態、締切状態を整える。
10. 自動テストと手動確認を行う。
11. Supabaseプロジェクト作成後に実接続確認を行う。
12. Vercel環境変数を設定し、デプロイ確認を行う。

## ステータス記法

- `[ ]`: 未着手
- `[~]`: 進行中
- `[x]`: 完了
- `[!]`: ブロック中
