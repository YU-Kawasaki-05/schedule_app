# Docs Summary

この `docs/` は予定調整アプリの要件、設計、実装計画、タスク、検証を小さな単位で管理するための入口です。

## フォルダ構成

| フォルダ | 内容 |
| --- | --- |
| `00_共通/` | プロジェクト横断の決定事項、用語、運用ルール |
| `01_要件定義/` | MVPスコープ、機能一覧、受け入れ条件、画面遷移 |
| `02_設計/` | アーキテクチャ、DB、API/Server Actions、UI設計 |
| `03_実装計画/` | 実装順序、フェーズ分割、リリース方針 |
| `04_運用検証/` | ローカル確認、Supabase設定、品質確認 |
| `05_タスク一覧/` | プロダクト完成までの工程別タスクリスト |

## 原文

- `要件.md`: 初期要件の原文。分割後の仕様は各フォルダ配下を正とし、原文は背景・意図の参照として残します。
